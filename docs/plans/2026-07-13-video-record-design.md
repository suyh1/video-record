## 1. 文档约束

本文是 video-record 的产品、交互、视觉、数据、架构、安全、运维和发布基线。

- **MUST**：必须实现或遵守，否则不得视为通过验收。
- **SHOULD**：默认遵守，偏离时必须在提交或 ADR 中说明理由。
- **MUST NOT**：明确禁止。
- 未写入本文的扩展能力不得自动进入 `v1.0.0`。
- 实施细节与本文冲突时，以本文为准；需要改变本文时先修改设计并重新确认。

## 2. 产品定义

video-record 是一套个人与家庭自托管的电影、电视剧观影档案。它以手动记录为完整主流程，通过 TMDB 提供搜索和基础元数据，并可选同步 Jellyfin、Emby、Plex 播放历史。

产品必须让用户在一分钟内完成常见记录，同时能通过时间、评分、标签、剧集进度和共同观看者回顾长期观影经历。

### 2.1 目标用户

- 默认体验面向单个自托管用户。
- 数据模型和权限从第一版起支持多个家庭成员。
- 每位成员独立拥有状态、评分、笔记、标签和片单。
- 一次观看事件可以关联多名共同观看者。

### 2.2 产品性格

克制、私密、可信。界面是一座安静的数字影库，具有电影质感，但不模拟影院，也不让装饰抢夺影视资料与个人记录的注意力。

### 2.3 v1 非目标

以下能力 MUST NOT 进入 `v1.0.0`：

- 求片、审批、下载、订阅、刮削和媒体文件管理。
- Radarr、Sonarr、下载器或站点管理。
- 公开社区、关注动态、热评榜和算法推荐。
- 原生移动应用和完整离线编辑。
- PostgreSQL、Redis、多实例部署和独立 worker。
- OAuth、第三方登录、2FA 和自动公网暴露。
- Webhook 实时同步、豆瓣抓取和公开分享海报。

## 3. 竞品研究与取舍

### 3.1 MoviePilot v2

研究来源：

- [MoviePilot](https://github.com/jxxghp/MoviePilot)
- [MoviePilot-Frontend](https://github.com/jxxghp/MoviePilot-Frontend)
- [MoviePilot Wiki](https://wiki.movie-pilot.org/)

结论：MoviePilot v2 是高密度 NAS 自动化控制台。video-record 只借鉴海报网格、桌面侧栏、全局搜索和任务状态反馈；MUST NOT 复制下载、订阅、整理、插件与运维配置密度。

### 3.2 Jellyseerr / Seerr

研究来源：

- [Seerr](https://seerr.dev/)
- [Seerr 文档](https://docs.seerr.dev/)
- [Seerr GitHub](https://github.com/seerr-team/seerr)

结论：借鉴其统一搜索、海报优先浏览、易扫读的电影/剧集详情和移动响应式能力。其“请求”主动作必须替换为“记录/更新记录”；季度请求、审批、请求限额和媒体可用性管理全部排除。

### 3.3 已阅

研究来源：[中国区 App Store，应用 ID 1589203887](https://apps.apple.com/cn/app/id1589203887)。

结论：采用想看/在看/看过的低成本记录闭环、日历回顾、月度/年度总结、重复观看、自定义条目和导入导出。最关键的质量标准是“每天一分钟完成记录”。

### 3.4 Apollo

研究来源：[中国区 App Store，应用 ID 1616467801](https://apps.apple.com/cn/app/id1616467801)。

结论：采用观看方式、弃看、二刷、标签统计和剧集进度双表达。长篇剧集必须同时显示季集编号与全剧累计集数。海报样式选择、重推荐和分享能力后置。

### 3.5 Filmo

研究来源：中国区/美国区 App Store，应用 ID `6741805793` 的公开产品信息。

结论：采用“共同观看者”语义和私人收藏叙事。一次事件可关联家庭成员，但不得替代任何成员的个人评分、笔记或状态。

### 3.6 综合方向

最终方向为：

> Seerr 的影视浏览清晰度 + 已阅的记录效率 + Apollo 的影视专业度 + Filmo 的家庭记忆。

MoviePilot 与 Seerr 的资源自动化、请求和运维域明确排除。

## 4. 信息架构

### 4.1 一级导航

一级导航固定为五项：

| 入口 | 内容 |
|---|---|
| 首页 | 继续观看、最近记录、近期回顾和快速记录 |
| 影库 | 想看、在看、看过、弃看、全部记录和个人片单 |
| 日历 | 按日期浏览电影、单集进度和重复观看事件 |
| 统计 | 月度、年度、类型、评分、时长、标签和观看方式 |
| 设置 | 账号、家庭成员、集成、导入导出、备份和系统状态 |

全局搜索是跨页面能力，MUST NOT 作为第六个一级页面。

### 4.2 搜索

- 桌面端 MUST 在顶部栏提供全局搜索。
- 移动端 MUST 在底部导航中提供明确的搜索入口。
- 搜索 MUST 同时返回本地缓存结果与 TMDB 结果并去重。
- 搜索项 MUST 显示电影/剧集类型、年份、原名和当前个人状态。
- 输入防抖固定为 `300ms`。
- 本地结果 MUST 先显示；TMDB 结果随后合并。

### 4.3 影视详情

详情页信息顺序固定为：

1. 海报、标题、年份、类型和基础元数据。
2. 主操作“记录”或“更新记录”。
3. 当前个人状态、评分、最近观看日期和剧集进度。
4. 观看历史时间线。
5. 个人笔记、标签和观看方式。
6. 剧集的季、集列表与进度。
7. 演职员、剧照等补充资料。

个人记录 MUST 位于演职员等外部资料之前。

### 4.4 首页约束

- 首页最多保留“继续观看”和“最近记录”两个横向区域。
- 其余内容 MUST 使用可扫描列表或海报网格。
- MUST NOT 构造流媒体式无限横向轨道。

## 5. 领域模型

### 5.1 影视身份

`media_items` 使用本地 UUID 作为内部身份。外部唯一约束为：

```text
source + source_id + media_type
```

电影与剧集共享基础表；`seasons`、`episodes` 使用独立表。业务外键 MUST NOT 直接使用 TMDB ID。

### 5.2 个人当前状态

`user_media_states` 保存成员对作品的当前聚合状态：

```text
none | wishlist | watching | completed | dropped
```

同时保存个人评分、私密笔记、开始日期、完成日期和版本号。

### 5.3 观看事件

`watch_events` 保存不可被聚合状态替代的事实：

- 开始/结束时间。
- 电影或具体单集。
- 观看方式。
- 记录来源。
- 完成度。
- 可选事件备注。

重复观看 MUST 新增事件，不得覆盖旧事件。

### 5.4 共同观看

`watch_event_participants` 将一次事件关联一名或多名成员。共享内容仅包括作品、时间、方式和参与者；成员个人状态、评分、标签和笔记继续独立保存。

### 5.5 剧集进度

`episode_progress` 按成员和单集保存已看状态与最近进度。系列进度由单集数据计算，界面 MUST 同时显示：

- 季集编号，例如 `S02E03`。
- 全剧累计集数。

### 5.6 评分

数据库统一使用 `0–100` 整数。界面默认使用 `0.0–10.0`。导入五星、十分和百分制数据时不得丢失原始可表达精度。

### 5.7 标签与片单

- `tags` 和 `user_media_tags` 默认用户私有。
- `collections` 和 `collection_items` 默认用户私有并支持手动排序。
- v1 MUST NOT 建立家庭全局标签词库或公开片单。

### 5.8 同步与审计

- `external_accounts`：媒体服务器配置引用。
- `external_media_mappings`：外部 ID 映射。
- `sync_runs`：游标、运行状态与结果摘要。
- `sync_candidates`：待确认匹配和冲突。
- `audit_events`：登录、成员、集成、导入、恢复和批量删除等高风险动作。

审计 MUST NOT 保存个人笔记正文、Authorization 头或任何凭据。

### 5.9 数据来源优先级

固定优先级为：

```text
用户手动 > 已确认导入 > 已确认同步 > 外部元数据默认值
```

低优先级来源 MUST NOT 覆盖高优先级字段。

## 6. 核心交互

### 6.1 首次启动

- 首次启动 MUST 进入封闭初始化页。
- 初始化 MUST 检测数据目录、迁移与 TMDB 配置状态。
- 创建首位管理员后 MUST 关闭公开初始化入口。
- v1 MUST NOT 开放自行注册；成员由管理员创建、停用或重置密码。

### 6.2 快速记录

表单分为“快速层”和“完整层”：

- 快速层默认只要求状态。
- 标记看过时默认创建日期为今天的事件。
- 评分、日期、观看方式、共同观看者、标签和笔记按需展开。
- 编辑已有记录时 MUST 直接显示已经填写的字段。

验收路径：

- 搜索并标记想看：不超过两次提交动作。
- 标记电影看过：默认日期为今天，评分和笔记可稍后补充。
- 首页推进下一集：一次只推进一集，并提供撤销。

### 6.3 电影规则

- `wishlist` 只更新状态，不创建事件。
- `completed` 和重看都创建新事件。
- 删除事件后重新计算首/末观看日期。
- 删除事件 MUST NOT 删除评分、笔记或标签。

### 6.4 剧集规则

- 支持单集、连续多集和整季标记。
- 默认观看日期为今天。
- 状态可以根据进度给出建议。
- 用户主动设置的 `dropped` MUST NOT 被自动改回。

### 6.5 同步候选

候选分为：精确匹配、可能匹配、无法匹配、冲突。

- 精确匹配且不存在更高优先级数据时 MAY 自动落档。
- 其他情况 MUST 由用户确认、重新匹配、忽略或创建自定义条目。
- 同一外部事件 ID MUST 只导入一次。

### 6.6 确认与撤销

以下操作使用可撤销提示，不弹确认框：

- 状态修改。
- 单集进度。
- 普通字段保存。

以下操作 MUST 二次确认：

- 删除观看事件。
- 清空个人记录。
- 删除成员。
- 断开集成。
- 恢复备份。

## 7. 异常处理

- TMDB 不可用时 MUST 保留搜索词、本地结果和已缓存详情。
- 上游限流 MUST 遵循重试时间与指数退避。
- 前端 MUST NOT 接收上游原始响应或凭据。
- 网络失败 MUST 保留表单输入。
- 字段错误 MUST 与字段关联，并将焦点移动到第一处错误。
- 写操作 MUST 使用幂等键。
- 可编辑聚合 MUST 使用版本号或 ETag。
- 冲突时 MUST 保留当前输入、显示服务器最新值并允许重新应用。
- 后台同步 MUST NOT 使用阻塞页面的全屏加载。

自定义条目只在 TMDB 无结果或同步无法匹配时出现；最少要求标题和媒体类型，年份可选。以后关联 TMDB 时 MUST 保留全部个人记录。

## 8. 视觉系统

### 8.1 使用场景与主题

用户会在夜间看完一集后用手机快速记录，也会在白天用桌面整理多年档案。因此主题默认跟随系统；亮暗主题必须功能等价。

### 8.2 色彩

策略为 Restrained。海报承担主要色彩，品牌色使用面积不得超过 10%。颜色只使用 OKLCH。

```css
:root {
  --bg: oklch(1 0 0);
  --surface: oklch(0.97 0.003 230);
  --ink: oklch(0.20 0.015 230);
  --muted: oklch(0.48 0.02 230);
  --border: oklch(0.88 0.008 230);
  --primary: oklch(0.54 0.15 230);
  --accent: oklch(0.34 0.11 25);
}

[data-theme="dark"] {
  --bg: oklch(0.09 0 0);
  --surface: oklch(0.14 0.006 230);
  --ink: oklch(0.94 0.005 230);
  --muted: oklch(0.68 0.012 230);
  --border: oklch(0.28 0.008 230);
}
```

关键对比度：

| 组合 | 对比度 |
|---|---:|
| 亮色正文/背景 | 约 `18.07:1` |
| 亮色次要文字/背景 | 约 `6.50:1` |
| 暗色正文/背景 | 约 `17.38:1` |
| 暗色次要文字/背景 | 约 `7.21:1` |
| 主按钮白字/主色 | 约 `4.66:1` |

状态 MUST 同时使用文字与图标：想看使用书签、在看使用播放/进度、看过使用勾选、弃看使用停止。颜色不得作为唯一状态信号。

### 8.3 排版与空间

- 自托管 Inter Variable，中文回退 Noto Sans SC。
- 数字使用 tabular-nums。
- 固定字号：`12/14/16/20/24/32px`。
- 正文：`14px` 或 `16px`。
- 页面标题不得超过 `32px`。
- 所有字距为 `0`。
- 间距：`4/8/12/16/24/32/48px`。
- 卡片圆角最大 `8px`，按钮和输入框 `6px`。
- 阴影只用于浮层，模糊不得超过 `8px`。

MUST NOT 使用渐变文字、装饰玻璃拟态、嵌套卡片、巨大指标模板、手绘假插图或装饰性大阴影。

### 8.4 海报

- 比例固定 `2:3`。
- 标题最多两行。
- 年份和状态保持一行。
- 缺图使用中性占位与片名字首。
- MUST NOT 生成虚构海报。

### 8.5 响应式

| 范围 | 布局 |
|---|---|
| `>=1280px` | 216px 侧栏、64px 顶栏、最大 1440px 内容区 |
| `768–1279px` | 72px 图标侧栏 |
| `<768px` | 56px 顶栏、64px 底部导航并适配安全区 |

影库网格：桌面海报最小 `160px`，平板 `144px`，移动双列且单项不小于 `136px`。

详情页：桌面为 `280px 海报 + minmax(0, 1fr)`，不得放入装饰卡；移动端 `132px` 海报与标题并排，主操作固定在安全区上方。

日历：桌面显示月网格；移动端显示按日议程。统计页使用无卡片分区与可访问图表。

### 8.6 组件和动态效果

每个交互组件 MUST 具备：`default/hover/focus/active/disabled/loading/error`。

- 加载使用保持尺寸的 skeleton。
- 空状态说明下一步并只提供一个主动作。
- 常规动效 `150–200ms`，抽屉 `220ms`。
- 动效只表达状态，使用 ease-out。
- MUST NOT 使用页面入场编排、弹跳或视差。
- `prefers-reduced-motion` 下移除位移和连续动画。

z-index 固定为：dropdown 20、sticky 30、backdrop 40、modal 50、toast 60、tooltip 70。

## 9. 技术架构

### 9.1 选型

- 后端：Go 模块化单体。
- 前端：React + Vite + TypeScript strict。
- 数据库：SQLite，单实例。
- 部署：单应用容器、单数据卷。
- v1 不使用 PostgreSQL、Redis 或独立 worker。

### 9.2 仓库结构

```text
cmd/server/
internal/
  auth/
  media/
  records/
  household/
  integrations/
  sync/
  stats/
  storage/
  httpapi/
web/
docs/
```

领域层 MUST NOT 依赖 HTTP、TMDB 或 SQLite 实现。

### 9.3 后端约束

- 使用标准 `net/http` 兼容路由或轻量 `chi`。
- SQLite 使用无 CGO 驱动。
- SQL 迁移嵌入二进制。
- 使用显式 repository 和类型化 SQL。
- MUST NOT 引入重量 ORM。

### 9.4 前端约束

- React Router。
- TanStack Query。
- React Hook Form + schema 校验。
- Radix 无样式原语。
- Lucide 图标。
- 项目 CSS 令牌。
- MUST NOT 引入整套带视觉意见的组件库。

### 9.5 API

- 前缀 `/api/v1`。
- JSON 使用 `camelCase`。
- 列表使用游标分页。
- 错误采用 RFC 9457 Problem Details。
- 错误包含稳定错误码和 `requestId`。
- 写操作使用 `Idempotency-Key`。
- 并发编辑使用版本号或 ETag。

## 10. 认证与凭据

### 10.1 认证

- 密码使用 Argon2id。
- 会话使用数据库保存的随机不透明 Token。
- Cookie 为 HttpOnly、SameSite=Lax；生产环境支持 Secure。
- 非 GET 同源请求校验 CSRF Token 与 Origin。
- 登录必须限流。
- 管理员可以撤销会话。

### 10.2 TMDB 凭据

- TMDB 令牌只允许通过服务端环境变量 `TMDB_READ_ACCESS_TOKEN` 注入。
- 真实值 MUST NOT 出现在源码、前端变量、Dockerfile、Compose 默认值、测试夹具、日志、错误响应、导出、备份或镜像层。
- 设置页只显示已配置/未配置和可选指纹，不回显令牌。
- 所有 Authorization 头在日志层统一脱敏。

### 10.3 媒体服务器凭据

- 使用 `APP_ENCRYPTION_KEY` 派生密钥进行 AEAD 加密。
- 数据库只保存密文、nonce、版本和指纹。
- 环境变量密钥不进入备份。
- 密钥缺失或更换时，集成进入锁定状态，不得损坏观影数据。

## 11. SQLite、缓存与同步

### 11.1 SQLite

- 启用 WAL、foreign_keys、busy_timeout。
- 限制单写、多读连接。
- 启动时先执行迁移。
- 迁移失败时 `/readyz` 不通过，业务端点不得开放。
- 同一数据卷只允许一个应用实例。

### 11.2 TMDB 缓存

- 搜索缓存 6 小时。
- 详情与季集快照缓存 7 天。
- 已记录条目使用 stale-while-revalidate。
- 海报默认使用 TMDB CDN。
- v1 不长期镜像全部海报。

产品设置或关于页 MUST 显示 TMDB 要求的归属说明。

### 11.3 媒体服务器同步

统一 `Provider` 接口，实施顺序为 Jellyfin、Emby、Plex。

- 默认每 15 分钟增量同步。
- 每日一次补偿同步。
- 任务、游标和结果必须持久化。
- 进程重启后补跑到期任务。
- 同步不得阻塞 HTTP 服务。

## 12. 健康、备份与恢复

- `/healthz` 只验证进程存活。
- `/readyz` 检查迁移、SQLite 可写和调度器初始化。
- TMDB 或媒体服务器短暂故障不得让服务整体不就绪。

目录固定为：

```text
/data/video-record.db
/data/backups/
```

系统备份 MUST 使用 SQLite Online Backup API 生成一致性快照，并附带 manifest 与校验和。MUST NOT 直接复制活动中的数据库和 WAL 文件。

恢复流程：

1. 校验版本、校验和和剩余空间。
2. 自动生成当前数据库快照。
3. 进入维护模式。
4. 原子替换数据库。
5. 失败时自动回滚。

升级前 MUST 自动备份。JSON/CSV 导出是数据迁移能力，不等同于系统备份。

## 13. Docker 与发布

### 13.1 镜像

- Node 阶段构建前端。
- Go 阶段关闭 CGO，生成静态二进制。
- 前端资源嵌入二进制。
- 最终使用非 root 精简 Debian 或 distroless 镜像。
- 镜像包含 CA 证书与时区数据。
- 容器只暴露 `8080`。
- `/data` 是唯一可写卷。
- 根文件系统支持只读。
- 固定非 root UID/GID。
- 启用 `no-new-privileges` 并 drop ALL capabilities。
- 二进制提供 `healthcheck` 子命令；不得为健康检查安装 curl。

### 13.2 本机开发

- Go 服务使用 `go run`。
- Vite 开发服务器代理 `/api`。
- 开发配置使用无真实值的 `.env.example`。
- 真实凭据只存在于被 Git 忽略的本机环境或安全密钥存储。

### 13.3 CI 与 Docker Hub

- Git 只使用 `main`。
- push `main` 执行完整验证。
- 推送 `vX.Y.Z` 标签后使用 Buildx/QEMU 构建并推送 `linux/amd64,linux/arm64` manifest。
- 发布生成 SBOM 和 provenance。
- 标签包含完整版本、主次版本和 `latest`。
- 只有正式 SemVer 标签可以更新 `latest`。
- Docker Hub 仓库由发布配置 `IMAGE_REPOSITORY` 明确指定，代码中不得写死个人命名空间。

## 14. v1 范围

`v1.0.0` MUST 包含：

- 封闭初始化、登录和成员管理。
- TMDB 搜索、详情和缓存。
- 电影和剧集详情。
- 五态个人影库。
- 评分、笔记、标签和片单。
- 观看事件、重看、共同观看。
- 单集进度。
- 日历与基础统计。
- 同步候选核对。
- Jellyfin、Emby、Plex。
- JSON/CSV 导入导出。
- 一致性备份恢复。
- 亮暗响应式 UI。
- Docker Hub 双架构镜像。

允许内部里程碑：

1. M1 基础设施与身份。
2. M2 TMDB 与手动记录。
3. M3 剧集、日历与统计。
4. M4 家庭、导入导出与备份。
5. M5 Jellyfin、Emby、Plex。

里程碑不得缩减公开 `v1.0.0` 范围。

## 15. 隐私

- 评分、笔记、标签和片单默认私有。
- 共同观看者可见共享事件的作品、日期和参与者。
- 用户可以主动向家庭公开评分或短评。
- 管理员在产品界面中无权读取其他成员私密笔记。

## 16. 测试与验收

### 16.1 单元测试

覆盖状态推导、事件幂等、评分转换、剧集进度、来源优先级、权限、缓存 TTL 与同步匹配。关键领域包行覆盖率不得低于 85%，但不得用覆盖率替代行为测试。

### 16.2 集成与契约

- 使用真实临时 SQLite。
- 使用伪造或录制的 TMDB、Jellyfin、Emby、Plex HTTP 合约。
- 覆盖全部迁移、N-1 升级、同步游标、重试、限流、凭据加解密与备份恢复往返。
- 测试夹具不得包含真实令牌。
- 每个 Provider 必须通过同一套 conformance tests。
- API 覆盖 RFC 9457、权限、游标、ETag、幂等和 CSRF。

### 16.3 前端与 E2E

- 组件测试覆盖表单、键盘、焦点、加载、空状态、错误和冲突。
- E2E 覆盖初始化、登录、搜索记录、重看、剧集推进、共同观看、同步候选、导出和恢复。

### 16.4 可访问性

- axe 自动检查。
- 完整键盘路径。
- 焦点顺序和屏幕阅读器名称。
- 200% 缩放。
- 非颜色状态。
- 亮暗对比。
- reduced-motion。
- 阻断级 WCAG 2.2 AA 违规必须为 0。

### 16.5 视觉回归

固定视口：

- `375x812`
- `768x1024`
- `1440x900`

每个视口覆盖亮暗主题，并检查横向溢出、遮挡、布局跳动、海报变形和按钮文字溢出。

### 16.6 性能

数据集：10,000 个影视条目、50,000 条观看事件、5 名成员。

| 指标 | 目标 |
|---|---:|
| 热缓存本地 API p95 | `<200ms` |
| 影库筛选 p95 | `<300ms` |
| 首页 LCP | `<2.5s` |
| 常用交互 INP | `<200ms` |
| CLS | `<0.1` |
| 应用对 TMDB 增加的处理开销 p95 | `<100ms` |
| TMDB 超时 | `8s` 后可恢复错误 |
| 10,000 条初次同步 | `<5min` |
| 100 条增量同步 | `<60s` |

同步期间 UI 必须保持可用。

### 16.7 容器、安全与可靠性

- amd64 与 arm64 分别执行启动、初始化、写入、重启持久化、健康检查和恢复。
- manifest 必须包含两个平台。
- 验证非 root、只读根文件系统和唯一可写数据卷。
- 执行 secret、依赖和镜像漏洞扫描。
- 覆盖限流、会话撤销、CSRF、越权、日志脱敏、恶意导入、路径穿越和恢复回滚。
- 高危和严重漏洞必须为 0。
- 写入、迁移、同步或备份中途终止后必须可恢复。
- 重复请求和重复同步不得产生重复事件。
- 恢复失败必须保留原数据库。
- 导出再导入后核心字段数量和值必须一致。

## 17. 发布门槛

仅在以下条件全部满足后，才可创建 `v1.0.0` 标签并更新 `latest`：

- v1 功能验收全部通过。
- amd64 与 arm64 镜像烟测通过。
- 备份恢复演练通过。
- WCAG 2.2 AA 阻断违规为 0。
- 高危与严重漏洞为 0。
- README、Compose、升级、备份、安全和凭据文档完整。
- secret scan 证明源码、历史、构建产物和镜像层不含真实凭据。

## 18. 设计变更流程

- 新功能先判断是否属于 v1 非目标。
- 涉及权限、个人数据、来源优先级、备份格式、API 兼容或部署拓扑的变更 MUST 写 ADR。
- 破坏备份、导出或 API 兼容性的变更 MUST 提供迁移路径。
- 视觉变更 MUST 在三种固定视口、两种主题和 reduced-motion 下验收。
- 本文发生实质变更时必须更新状态、日期与对应实施计划。
