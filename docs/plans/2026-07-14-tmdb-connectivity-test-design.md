# TMDB 连通性测试设计

## 目标

在设置页“元数据来源”区域的“TMDB 已配置”状态右侧增加“测试连通”按钮。用户点击后，应用从服务端发起一次不使用缓存的 TMDB 请求，以验证令牌、服务端网络和代理链路当前确实可用。

## 交互

- 仅在 TMDB 已配置时显示“测试连通”按钮。
- 请求期间禁用按钮并显示“正在测试”，避免重复请求。
- 成功后在同一区域显示“TMDB 连通正常”。
- 失败时显示可操作的具体原因：令牌无效、请求超时、请求受限，或“无法连接 TMDB，请检查服务端代理或网络设置”。
- 结果使用可访问的状态或警告语义，并在窄屏下允许状态、按钮和结果自然换行。

## 服务端设计

新增受登录保护的 `GET /api/v1/tmdb/connectivity` 端点。端点调用 TMDB `/configuration`，该请求：

- 携带现有服务端 Bearer token；
- 不读取或写入 TMDB 缓存；
- 使用现有超时设置和 HTTP client，因此遵循 Go 默认 transport 支持的 `HTTP_PROXY`、`HTTPS_PROXY` 与 `NO_PROXY`；
- 成功时只返回 `{ "connected": true }`；
- 不返回令牌、上游响应正文或远端地址。

TMDB 客户端将 401/403 映射为独立的未授权错误；429、超时及其他网络或上游异常继续使用稳定错误类型。HTTP 层将其转换为现有 Problem 响应风格，供前端显示对应中文提示。

## 前端设计

API client 新增连通性测试方法。`TmdbStatus` 使用 React Query mutation 管理请求状态，并在现有状态标签右侧渲染按钮。错误提示按稳定 Problem code 映射，不展示原始错误内容。

此功能不增加代理配置表单。部署者继续通过服务端进程或容器环境配置出站代理。

## 测试

- TMDB 客户端测试验证探测请求命中 `/configuration`、携带令牌且不会命中缓存。
- 客户端测试验证 401/403 与网络、超时、限流错误可区分。
- HTTP handler 测试验证成功响应、稳定错误码和无凭据泄漏。
- 前端组件测试验证按钮仅在已配置时出现、加载状态、成功反馈和各类失败提示。
- OpenAPI 契约和生成类型保持同步，并运行 Go、前端单测、类型检查与 lint。
