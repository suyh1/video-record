# 多刷详情单集标题修复设计

## 问题

当前季观看列表通过 TMDB 实时季目录展示单集标题，而本地只为已记录单集保存最小身份。归档详情直接读取本地 `episodes.name`，因此稀疏身份的空名称在界面中显示为“未命名”。

## 已确认方案

`SeasonRecordWorkspace` 将当前季目录传给同一页面内的 `RewatchSection`。归档弹窗按季号和集号匹配目录条目，标题按以下顺序展示：

1. 当前季 TMDB 目录标题；
2. 归档接口返回的本地标题；
3. “未命名”兜底。

电影没有分集目录，保持现有逻辑。归档的观看时间、评分、观看方式和私人笔记仍来自只读归档详情，不改变后端契约和稀疏存储边界。

## 数据流

1. `SeasonRecordWorkspace` 已通过 `tmdbId` 和当前季号确定活动季。
2. 当前季的 `EpisodeProgress` 与 `RewatchSection` 共享 React Query 中的 TMDB season 查询结果。
3. `RewatchSection` 只接收用于展示的分集目录，并按 `seasonNumber + episodeNumber` 建立标题查找。
4. 用户打开某一刷详情时，归档条目继续提供准确观看时间，目录仅补充标题。

## 验证

- 组件回归：归档接口返回空名称、目录提供标题时，弹窗展示目录标题且不显示“未命名”。
- 兼容回归：电影归档仍不显示分集列表；目录缺失或标题为空时保留原有兜底。
- 页面回归：活动季切换后，多刷详情使用对应季目录，不串季。
