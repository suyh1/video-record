# 影视详情与记录页改造设计

**日期：** 2026-07-14  
**状态：** 已确认  
**分支：** `main`

## 目标

把现有详情页改造成充分利用宽屏、以影片画面和个人记录为核心的页面；补齐演员展示和真正可用的剧集季/集记录，同时避免把 TMDB 的完整目录复制进本地业务数据库。

## 已确认原则

- 本地数据库只持久化本项目产生或必须拥有的数据。
- TMDB 的 backdrop、简介、类型、演职员、季列表和分集资料在页面加载时获取。
- TMDB 数据可进入现有带过期时间的短期缓存，但不写入 `media_credits`，也不批量写入 `seasons` 与 `episodes`。
- 本地保留影视与 TMDB 的身份关联，以及标题、年份和海报路径这组很小的离线兜底快照。
- 用户真正产生的观看状态、评分、笔记、标签、片单、观看事件和已看分集必须可靠持久化。
- TMDB 不可用不能阻止读取或修改已有个人记录。

## 方案比较

### 方案一：首次入库时复制完整 TMDB 目录

实现直接，现有剧集进度查询可继续依赖本地 `seasons`/`episodes`。但每部剧会产生大量从未观看的季集行，资料会过期，还需要持续同步和删除冲突处理。该方案不采用。

### 方案二：每次打开都绕过缓存访问 TMDB

数据最及时，但页面容易被上游延迟、故障和限流拖慢；同一家庭的重复访问也会产生没有必要的请求。该方案不采用。

### 方案三：按需获取并使用短期缓存

页面按需请求剧集、季和演员；服务端使用 6 小时 TTL 缓存 TMDB 响应。本地只保存用户标记过的分集身份和进度。该方案兼顾数据新鲜度、数据库体积与故障降级，确定采用。

## 数据边界

### 本地业务数据

- 本地媒体 ID、媒体类型和 TMDB 外部 ID。
- 标题、年份和海报路径的最小离线兜底快照。
- 用户状态、评分、私人笔记、标签和片单关系。
- 观看事件、观看时间、来源、方式和共同观看者。
- 用户实际标记过的 TMDB 分集 ID、季号、集号和观看进度。
- 自定义影视及用户录入的内容。

### 按需 TMDB 数据

- backdrop、完整简介、时长、类型和其他展示元数据。
- 演员姓名、角色、排序和头像路径。
- 季名称、季海报、首播日期和总集数。
- 分集名称、简介、剧照、播出日期和时长。

这些数据只通过服务端适配器返回。现有 `tmdb_cache` 继续作为可过期的传输缓存，不成为导出、备份语义中的业务记录。

## TMDB 读取与新鲜度

- 电影详情、剧集详情、季详情和 credits 使用 6 小时缓存 TTL。
- 页面加载时立即显示本地兜底信息和个人记录，同时独立请求 TMDB 详情、演员和当前季。
- 缓存未过期时直接返回；过期后的首次请求重新访问 TMDB，随后覆盖缓存。
- 新季通过剧集详情中的季摘要出现；新集通过所选季详情出现，最长约 6 小时可见。
- 用户切换季时才加载该季分集，不预取并落库整部剧的全部分集。
- TMDB 请求失败时保留本地头部和个人记录；演员或季集区域显示局部错误与重试，不让整页白屏。

## 分集进度模型

现有问题不是缺少交互，而是进度读取把完整分集目录当成本地业务数据。改造后目录与进度分离：

1. 剧集详情请求返回远端季摘要。
2. 当前季请求返回远端分集目录。
3. 本地进度请求只返回用户已经标记过的季号、集号、TMDB 分集 ID和观看时间。
4. 前端按 TMDB 分集 ID，必要时回退到季号与集号，把远端目录和本地进度合并。
5. 用户标记单集、范围、整季或下一集时，服务端只为被标记的分集保存最小身份行和项目进度，不保存标题、简介、剧照等 TMDB 展示字段。
6. 撤销继续删除对应进度和由该进度产生的观看事件；其他评分、笔记和标签不受影响。

已有本地季集数据需要兼容读取。迁移保留已有观看进度和观看事件，新接口不再依赖未观看分集是否已经落库。

## 服务端接口

调整以下受会话保护的接口：

- `GET /api/v1/media/{id}`：在本地兜底资料中增加可空的 `tmdbId`，供前端选择性加载外部资料。
- `GET /api/v1/tmdb/movie/{id}` 与 `GET /api/v1/tmdb/tv/{id}`：复用现有代理并扩展完整展示资料；剧集响应包含季摘要。
- `GET /api/v1/tmdb/{mediaType}/{id}/credits`：新增主要演员读取；不持久化演员。
- `GET /api/v1/tmdb/tv/{id}/season/{seasonNumber}`：扩展现有单季响应，按需返回该季分集资料。
- `GET /api/v1/records/{mediaID}/progress`：返回本地已看分集引用、版本、状态和计数，不再要求本地存在完整目录。
- `POST /api/v1/records/{mediaID}/progress`：接受由当前远端目录选择出的分集引用，继续使用 CSRF、幂等键与乐观版本控制。

服务端只接受与当前媒体类型和 TMDB 身份匹配的请求。输入严格校验正整数季号、集号和外部分集 ID；普通用户只能修改自己的进度。

## 页面结构

### 影视头部

- 在现有 1440px 主内容宽度内使用全宽 backdrop 区域，不增加第二套应用导航。
- backdrop 为真实 TMDB 图片，并叠加保证文本可读的中性色遮罩。
- 左侧叠放 2:3 海报；右侧显示媒体类型、标题、原名、年份、类型、时长、个人评分和简介。
- 没有 backdrop 时使用现有表面色和海报主视觉，不使用装饰性渐变或占位图库。
- 本地自定义影视继续显示“关联 TMDB”入口。

### 演员

- 头部下方展示主要演员横向列表。
- 每项包含 2:3 头像、姓名和角色；使用稳定尺寸，横向滚动不改变页面宽度。
- 没有头像时使用一致的姓名占位，不请求第三方随机图片。
- 没有 TMDB 身份、无演员数据或 TMDB 错误时使用独立空态/错误态。

### 主体双栏

- 桌面采用 `minmax(0, 1fr) / 400px` 双栏。
- 左栏依次放剧集进度（仅剧集）和观看历史。
- 右栏放个人记录，允许在宽屏中保持可见但不遮挡底部导航。
- 小于平板宽度时变为单栏，个人记录排在剧集进度与历史之前。

### 剧集进度

- 顶部显示已看集数、总集数和“下一集”动作。
- 使用季选择器，一次只展示一个季，默认定位到最后观看季或第一个未完成季。
- 分集按稳定行或紧凑网格展示集号、标题、播出状态和已看状态。
- 单集按钮可直接切换；范围与整季记录放入“批量记录”折叠区。
- 当前季加载时使用结构化 skeleton；失败时显示“重新获取分集资料”。

### 家庭与整理

- 私人标签、片单、家庭公开设置和家庭评价移入默认收起的“家庭与整理”区域。
- 观看状态、评分、笔记和保存记录保持在主记录界面。
- 折叠使用原生 `details/summary`，保留键盘操作和明确焦点。

## 响应式与无障碍

- 海报、演员头像、季选择器和分集按钮使用稳定尺寸，加载和状态变化不造成布局跳动。
- 375px、768px、1440px 和宽桌面均不得出现横向页面溢出。
- 背景图上的文字满足 WCAG 2.2 AA；图片有准确替代文本。
- 分集状态不能只依赖颜色，必须有图标、文本或可访问名称。
- 所有自动 TMDB 请求尊重现有错误边界，不抢夺焦点或用 toast 打断阅读。
- `prefers-reduced-motion` 下不添加非必要运动。

## 错误处理

- 外部详情失败：继续显示本地标题、年份、海报和个人记录。
- credits 失败：演员区域单独重试。
- 季详情失败：当前季区域单独重试，已保存进度仍可作为文字摘要显示。
- 进度保存失败：保留当前选择并显示行内错误；版本冲突重新读取本地进度后允许重试。
- TMDB 未配置：隐藏不可执行的丰富资料请求，显示设置状态，不影响本地记录。

## 测试策略

### Go

- TMDB TV、season 和 credits 的 6 小时缓存与错误映射。
- 本地媒体 ID 到 TMDB 身份的隔离和无身份错误。
- 只保存被用户标记的分集身份，不批量写入远端目录。
- 单集、下一集、范围、整季、撤销和幂等行为。
- 新增远端季集在缓存过期后可见，已有进度不丢失。
- TMDB 故障时本地记录读取和写入边界不受影响。

### React

- backdrop 头部、本地兜底和外部详情增强。
- 演员加载、空态、无头像与错误重试。
- 季切换、逐集切换、下一集、范围和整季记录。
- 家庭与整理默认收起，展开后原控件仍可用。
- 外部请求失败时个人记录仍可见、可保存。

### 浏览器

- 使用真实本地 API 和合成 TMDB 上游验证完整剧集流程。
- 验证 1440x900、768x1024、375x812 及宽桌面布局。
- 检查横向溢出、固定元素遮挡、键盘焦点、axe 和控制台错误。
- 对浅色和深色模式分别保存详情页视觉回归截图。

## 不在本次范围

- 常驻后台 TMDB 爬取或全库定时同步。
- 本地完整演职员库、人物详情页或人物搜索。
- 删除所有历史 TMDB 快照与重写全站影库数据模型。
- 流媒体播放入口、预告片、相似影视或社交功能。
