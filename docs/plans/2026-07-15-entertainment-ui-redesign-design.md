# 影视娱乐化 UI 改造设计

**日期：** 2026-07-15
**状态：** 已确认
**分支：** `main`

## 目标

把 video-record 从偏管理后台的界面升级为以影视画面为主角的个人与家庭观影档案，同时保留现有记录流程、响应式布局、亮暗主题和 WCAG 2.2 AA 核心要求。

本次采用“海报驱动型”方向：应用框架保持中性，当前影视 backdrop 提供页面氛围色。娱乐感主要来自真实 TMDB 图片、影视标题层级、海报布局和克制动效，而不是固定的蓝紫渐变、霓虹装饰或复杂 3D 效果。

## 已确认原则

- 在当前 `main` 分支上分批实施，不迁移 React、Vite、原生 CSS token 或 Go HTTP API 技术栈。
- 登录页直接依赖 Compose 中配置的 TMDB token 与代理，轮播 TMDB 热门电影和热门电视剧。
- 不在项目中内置精选影视图片，不使用库存图片、虚构海报或渐变图作为图片失败兜底。
- TMDB API 与图片都由 video-record 后端访问；浏览器不直接请求 `api.themoviedb.org` 或 `image.tmdb.org`。
- 登录页或首页没有任何成功加载的背景图时使用纯白背景，登录、搜索和记录功能保持可用。
- 动态取色只用于环境光、进度条、选中边缘和细小高亮，不改变正文颜色，不让整个应用随海报频繁变色。
- 先建立公共数据边界和基础视觉体系，再依次改造登录、首页、影库、详情及辅助页面。
- 保留浅色、深色、键盘操作和减少动态效果支持。
- 不改变观影记录、家庭共享、备份恢复、同步或 TMDB 业务数据模型。

## 方案比较

### 方案一：基础体系先行，页面分批改造

先完成公开热门接口、图片代理、视觉 token、按钮和表单，再按页面逐批改造。该方案视觉一致、提交范围可控，每批均可独立测试，确定采用。

### 方案二：登录页与首页优先

能够最快产生明显效果，但前两页会先形成临时样式，公共控件后续需要二次整理。该方案不采用。

### 方案三：全站一次性重构

最终视觉统一，但改动、视觉快照和响应式回归集中发生，不适合当前 `v1.0.0` 验收阶段。该方案不采用。

## TMDB 热门内容

### 公开接口

新增未登录可读的 `GET /api/v1/public/tmdb/highlights`。接口分别请求 TMDB：

- `/movie/popular?language=zh-CN&page=1`
- `/tv/popular?language=zh-CN&page=1`

服务端过滤没有 `backdrop_path` 的项目，按照电影、剧集交替排列，默认返回 10 项。响应只包含轮播所需的最小字段：

- TMDB ID
- `movie` 或 `tv` 类型
- 中文标题
- 原始标题
- 年份
- 简介
- 指向本站的签名 backdrop URL

热门列表使用现有 `tmdb_cache`，缓存 TTL 为 6 小时。TMDB 未配置、超时、限流或代理不可用时返回稳定问题响应；登录页和首页把这些错误视为可降级的背景数据错误。

现有搜索、详情、演员、季和集接口继续位于 `Authenticate` 中间件之后。公开范围只包含热门列表及其签名图片，不允许未登录用户调用搜索或访问私人记录。

### 图片代理

新增未登录可读的签名图片代理。热门接口返回本站 URL，浏览器不接触 TMDB token，也不直接访问 TMDB 图片域名。

签名覆盖图片路径、尺寸和有效期。后端使用服务端密钥验证签名，拒绝篡改的路径、过期 URL 和未允许的尺寸。第一批只开放登录与首页 backdrop 使用的 `w1280`；影库和详情改造时扩展到：

- `w342` 海报
- `w300` 演员头像
- `w780` 剧照
- `w1280` backdrop

代理只能拼接固定的 `https://image.tmdb.org/t/p/` 上游地址，不能接受任意 URL。上游请求复用 TMDB 客户端的超时与 Compose 代理环境，并限制：

- 请求方法为 `GET`
- 路径格式与扩展名
- 响应 Content-Type
- 最大响应体积
- 上游超时
- 并发与取消传播

成功响应设置公开浏览器缓存头，减少重复下载，但图片不永久写入项目数据目录。上游失败返回稳定 HTTP 错误，不返回本地替代图片。

## 全局视觉体系

### 颜色与表面

- 浅色模式使用白色画布、冷灰表面和接近黑色的正文。
- 深色模式使用偏黑中性灰，不使用大面积蓝紫色。
- 品牌标记保留胶片红，作为没有影视图片时的默认强调色。
- 有 backdrop 的页面从同源代理图片中提取一个受控主色，写入 `--media-accent`。
- 取色忽略接近纯黑、纯白和低饱和像素，并限制最终亮度与饱和度范围。
- 动态色不直接承担正文可读性；不合格或提取失败时回落品牌红。
- 卡片只用于重复内容、模态框和真正需要边界的记录工具，不为每个页面区段增加浮动卡片。

### 字体

- 中文正文优先使用 `Noto Sans SC`、苹方、微软雅黑和系统无衬线回退。
- 品牌、英文标题和数字使用本地打包的可变展示字体，不依赖外部字体 CDN。
- 影视标题、页面标题、章节标题、元数据和辅助信息形成明确层级。
- 不使用负字距；长标题允许平衡换行；统计数字使用等宽数字特性。

### 按钮

建立主按钮、次按钮、文字按钮和图标按钮四级体系。所有按钮使用稳定高度、统一图标尺寸和 6px 圆角，并覆盖：

- 悬停抬升或颜色变化
- 按压反馈
- 加载状态
- 禁用状态
- 键盘焦点
- 稳定宽高，状态变化不引发布局跳动

熟悉的工具动作优先使用现有 Lucide 图标；不为关闭、搜索、显示密码等动作使用冗长文本按钮。未知图标提供可访问名称和悬停说明。

### 表单

输入框、选择器、文本域和分段控件共享标签、辅助说明、错误信息、高度、边框与焦点规则。评分和观看状态保留适合其语义的可视控件，不强行改成普通下拉框。

错误在字段或表单内部显示；保存成功就地反馈；不使用 `window.alert()`，不因错误清空用户已经输入的内容。

### 动效

- 120–160ms：按钮、图标和焦点反馈。
- 200–280ms：菜单、筛选、海报信息层和局部进入。
- 约 800ms：登录页和首页 backdrop 交叉淡入。

位移动效只使用 `transform` 与 `opacity`。不增加滚动劫持、惯性滚动或大型动效库。`prefers-reduced-motion` 下关闭自动轮播、推进、抬升和非必要分批进入。

## 登录与初始化

### 桌面布局

登录页使用全视口 TMDB backdrop，表单面板居中。背景使用 `cover`，并加中性遮罩保证操作层稳定可读。登录面板约 420px 宽、8px 圆角，使用半透明白色表面和克制的边缘折射，不使用紫色渐变玻璃。

内容顺序为品牌标记、`video-record`、欢迎文字、用户名、密码、行内错误和登录按钮。输入框有始终可见的标签和图标；密码框增加显示或隐藏按钮。加载只替换按钮图标与文字，不改变尺寸。

### 轮播行为

- 每 7 秒自动切换。
- 使用约 800ms 交叉淡入和很轻的镜头推进。
- 只预加载当前图片与下一张图片。
- 第一张使用高优先级加载。
- 图片完成解码后才加入轮播。
- 单张失败时尝试下一张；所有图片失败时显示纯白背景。
- `prefers-reduced-motion` 下取消自动切换和推进，只显示第一张成功图片。
- 页面不可见时停止计时，恢复可见后重新计时。

热门列表和图片加载不能阻塞用户名、密码与登录提交。TMDB 不可用时表单立即在白色背景上可用。

### 其他认证状态

管理员初始化、状态检查和连接失败页复用同一背景与表面体系。初始化面板适当加宽以容纳服务状态与三个输入框，但不改变初始化关闭、密码规则或凭据提交逻辑。

移动端保留全屏背景，提高操作层不透明度并占据大部分可用宽度。面板略低于视觉中心，避免影视主体、软键盘和安全区域遮挡表单。

## 应用框架与导航

桌面移除固定左侧栏，改用顶部影视应用导航。导航包含品牌、首页、影库、日历、统计、搜索、记录和设置。

首页和详情页初始状态下，导航覆盖在主视觉上方；滚动后切换为带轻微模糊的中性实色背景。普通页面直接使用实色导航。导航高度固定，页面内容为它预留空间，不依赖任意高 `z-index` 修补重叠。

移动端继续使用底部导航，顶部只保留品牌、搜索和记录动作。底栏适配安全区域，不遮挡最后一项内容或表单提交按钮。

## 首页

### 个性化主视觉

首页优先从正在观看与最近记录中选择具有 TMDB backdrop 的影视，最多 6 项。每项显示标题、年份、类型、观看进度和一个主操作，例如“继续第 2 季第 4 集”或“查看记录”。

个人记录没有可用 backdrop 时，使用公开热门列表补充。所有个性化与热门图片都通过本站代理；全部失败时使用纯白背景与深色文字。

桌面主视觉高度约为当前视口的 65%–72%，同时让下一段“继续观看”在首屏底部露出。移动端缩短主视觉、简介和操作数量，保持标题与按钮不溢出。

轮播每 8 秒切换，提供上一张、下一张、进度指示与暂停语义。用户手动操作后暂停自动切换；页面不可见时停止；减少动态效果模式下只显示第一项。

### 继续观看

使用横向海报轨道，展示季集进度、下一集名称和进度条。桌面悬停显示快速继续操作，但标题与进度不依赖悬停才能读取。没有在看内容时显示一个明确搜索入口。

### 最近记录

使用一张较大的最近记录和若干紧凑条目组合，展示观看状态、评分、观看日期和重看次数。点击条目进入详情，不在首页复制完整编辑表单。

轮播详情、继续观看与最近记录独立加载和报错，一个区域失败不能让整个首页白屏。

## 影库与搜索

影库使用响应式海报墙：桌面 5–7 列，平板 3–4 列，手机固定 2 列。卡片以 2:3 海报为主体，不增加厚重边框；标题、年份、类型、状态和观看进度位于稳定位置。

桌面悬停显示记录状态与快速查看，移动端始终显示必要信息。海报失败时显示纯白或当前中性表面和标题首字，不显示破图图标或模拟图片。

状态筛选使用横向分段控件。个人片单改成紧凑片单栏，创建片单通过行内展开或侧边面板完成，不长期占据影库顶部。筛选栏滚动后固定在顶部导航下方。

搜索弹层区分本地记录和 TMDB 结果，支持最近搜索、结构化 skeleton、键盘上下选择与回车确认。搜索结果图片、标题和元数据使用稳定尺寸。

## 影视详情

详情页继续使用现有业务数据与记录组件，视觉改造包括：

- backdrop 全宽延伸至顶部导航下方。
- 海报与标题区形成叠层，集中展示标题、原名、年份、类型、时长和评分。
- 演员使用稳定尺寸的横向轨道。
- 剧集进度、观看历史和个人记录继续采用高效双栏工作区。
- 继续观看与保存记录获得主操作权重。
- 家庭与整理等低频功能继续折叠。
- backdrop 失败时使用中性纯色头部，不生成装饰图。

详情页不自动轮播，不用氛围效果干扰记录和批量剧集操作。

## 日历、统计与设置

日历桌面保留月历与日程联动，使用清晰的当前日期、已观看日期和选中日期状态。移动端优先展示日程，月历压缩为可切换区域。

统计页通过更有张力的数字层级展示观看数量、时长、类型分布和趋势。图表保留文字数据，不使用纯装饰 3D 图表，不让动态色破坏系列区分。

设置页使用顶部章节导航，分为账户、TMDB 与媒体服务器、家庭成员、数据导入导出、备份与恢复。每组使用标题、说明与分隔线组织，不嵌套多层卡片。危险操作保留独立区域和明确确认流程。

新增包含返回首页与搜索入口的品牌化 404 页面，避免未知路由成为空白页。

## 加载、空状态与错误

- skeleton 与最终内容尺寸一致，避免布局偏移。
- 空状态使用品牌标记、简短说明和一个明确动作，不使用库存插图。
- 外部资料错误在对应区域重试，不影响已有本地记录。
- 图片错误使用纯白或当前中性表面，不显示破图图标。
- 保存错误保留当前选择与输入；版本冲突沿用现有重新读取和重试语义。
- 保存成功就地反馈，不使用打断操作的模态框。
- 登录页与首页不能因为 TMDB 或图片失败而阻止核心操作。

## 响应式与无障碍

以 375×812、768×1024、1440×900 和宽桌面作为固定验证尺寸。

- 海报、按钮、轮播、导航和计数器使用稳定尺寸。
- 页面不得出现横向溢出。
- 顶部导航、移动底栏、软键盘和安全区域不得遮挡内容或操作。
- 自动轮播提供上一张、下一张和暂停状态。
- 纯背景图片为装饰，不重复朗读影视标题。
- 图片文字区域满足 WCAG 2.2 AA 对比度。
- 键盘可完成登录、搜索、筛选、记录和剧集操作。
- 所有交互焦点始终可见。
- 状态不能只依赖颜色表达。
- 减少动态效果模式关闭自动轮播和非必要位移。

## 分批交付

### 第一批：TMDB 公共能力

- 热门电影、热门剧集、缓存与归一化。
- 签名图片 URL、尺寸白名单、后端代理、超时和响应限制。
- OpenAPI、生成类型和前端请求函数。

### 第二批：基础视觉与登录页

- 颜色、字体、按钮、表单和动效 token。
- 登录、初始化、状态检查和连接失败页。
- TMDB 轮播、白底降级与减少动态效果。

### 第三批：应用框架与首页

- 桌面顶部导航和移动底栏。
- 个性化首页轮播、继续观看和最近记录。
- 个人内容不足时使用热门内容补充。

### 第四批：影库、搜索与详情

- 海报墙、筛选、片单栏和搜索弹层。
- 详情头部、演员轨道和记录工作区。
- 所有 TMDB 图片迁移到后端代理。

### 第五批：辅助页面与全站验收

- 日历、统计、设置和 404。
- 加载、空状态、错误状态和响应式收尾。
- 全量视觉回归与无障碍检查。

## 测试策略

### Go

- 热门电影与剧集请求、过滤、交替合并、6 小时缓存和错误映射。
- 热门接口无需登录，其他 TMDB 与私人接口仍返回未认证。
- 签名有效、过期、篡改、非法尺寸、非法路径、超大响应和错误 Content-Type。
- 图片请求使用服务端上游客户端并传播取消与超时。
- 图片响应缓存头和稳定错误响应。

### React

- 登录热门列表加载、轮播切换、手动暂停、图片解码失败和纯白降级。
- 密码显示、提交加载、错误保留和初始化状态。
- 首页个人内容优先级、热门补充、动作和局部错误。
- 顶部导航滚动态、移动导航和未知路由。
- 海报墙、筛选、搜索键盘操作、详情图片失败和动态色回落。
- 减少动态效果下不自动切换。

### 浏览器

- 登录、首页、影库和详情页的 375×812、768×1024、1440×900 浅色与深色视觉快照。
- 浏览器网络中不存在对 `image.tmdb.org` 的直接请求。
- 无横向溢出、导航遮挡、布局跳动和控制台错误。
- 键盘主流程和 axe 核心流程检查。
- 图片失败、TMDB 失败、空影库和长标题场景。

每一批运行相关 Go 测试、Vitest、类型检查、构建和 E2E；不把验证集中到最后一批。

## 不在本次范围

- 预告片或正片播放。
- 推荐算法或相似影视系统。
- TMDB 用户账户登录。
- 外部字体 CDN。
- 3D 场景、滚动劫持或大型动效库。
- 本地精选背景图或库存图片兜底。
- 改写现有观影记录与家庭共享数据模型。
