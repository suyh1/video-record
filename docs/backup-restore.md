# 备份与恢复

## 备份内容

管理员在“设置 -> 备份与恢复”创建系统备份。应用使用 SQLite Online Backup API 生成一致性快照，并输出 `.vrbackup` 归档。归档固定包含：

```text
manifest.json
video-record.db
```

manifest 记录格式版本、数据库架构版本、迁移校验和、字节数与 SHA-256。归档不包含环境变量，也不包含 `APP_ENCRYPTION_KEY` 或 TMDB token。

数据库包含观影记录、家庭数据和使用 AEAD 加密后的媒体服务器凭据。备份归档本身不是额外加密容器，因此必须按敏感个人数据保护。

## 备份步骤

1. 使用管理员账户打开“设置 -> 备份与恢复”。
2. 选择“创建系统备份”。
3. 从备份列表下载新生成的 `.vrbackup`。
4. 把归档复制到独立于应用主机和 Docker volume 的加密存储。
5. 同时确认 `APP_ENCRYPTION_KEY` 已保存在独立密码管理器或离线密钥库。

`/data/backups/` 中的服务器副本便于快速恢复，但它与主数据库位于同一 volume，不能替代异地备份。

建议至少保留：最近 7 个每日备份、4 个每周备份，以及每次升级前的一份备份。具体保留期应按家庭隐私和可用磁盘调整。

## 恢复步骤

恢复是管理员专用破坏性操作，会用归档内数据库覆盖当前数据：

1. 确认 `.vrbackup` 来自可信来源，并保留当前数据的独立备份。
2. 打开“设置 -> 备份与恢复”，选择归档。
3. 在确认对话框核对文件名，选择“确认恢复”。
4. 等待维护模式结束；必要时重新登录。
5. 验证家庭成员、影库条目、观看事件和集成状态。

应用会校验归档结构、checksum、迁移历史、架构版本、可用空间和 SQLite 完整性。替换前会自动保存当前数据库快照；任何校验、重开或提交失败都会保留或回滚到原数据库。

## 恢复演练

每次正式发布前执行一次使用合成数据的 backup/restore 演练：

1. 创建名为“恢复演练基线”的自定义影视记录并保存一次观看事件。
2. 创建并下载系统备份，记录归档文件名和 SHA-256。
3. 在备份后创建名为“恢复后应消失”的第二条合成记录。
4. 恢复刚才的归档。
5. 确认“恢复演练基线”及观看事件仍存在，第二条记录不存在。
6. 检查 `/readyz`、登录、影库和备份列表。
7. 删除演练数据或恢复演练前的正式备份。

不要使用真实凭据或私人笔记作为演练标记。

## 加密密钥保留

`APP_ENCRYPTION_KEY` 不进入备份，这是刻意的安全边界。恢复到另一台主机时必须提供原密钥，才能解密 Jellyfin、Emby、Plex 凭据。

丢失或更换 `APP_ENCRYPTION_KEY` 会锁定媒体服务器集成，但不会损坏观影数据；已有观影记录、评分、笔记、标签、日历和统计仍可访问。恢复结果会显示 `integrations_locked` 警告，直到重新提供原密钥或重新连接集成。

不要为了消除锁定警告反复尝试未知密钥，也不要直接编辑 SQLite 中的凭据列。

## 主机迁移

1. 在旧主机下载最新 `.vrbackup`。
2. 从安全密钥库取出原 `APP_ENCRYPTION_KEY`，不要把它放进备份归档。
3. 在新主机按部署文档启动相同或更新版本。
4. 完成首位管理员初始化后，从设置页恢复归档。
5. 配置原密钥并验证集成未锁定。
6. 验证完成前保留旧主机且不要重用其写入服务。

升级的可回滚冷卷快照见[升级与回滚](upgrading.md)。
