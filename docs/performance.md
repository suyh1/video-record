# 性能与故障恢复基线

本文记录 `v1.0.0` 本地性能目标、合成数据集、测量方法和进程中断恢复验证。测试不访问 TMDB 或媒体服务器，也不读取或保存真实凭据。

## 运行方式

```bash
./scripts/perf-smoke.sh
./scripts/recovery-smoke.sh
```

性能脚本最长允许十分钟完成一次运行。恢复脚本为每个中断点创建独立的临时 SQLite 数据库，并在测试结束后删除。

## 基准环境

- 日期：2026-07-13
- 系统：macOS 26.5.2，arm64
- 处理器：Apple M2 Max
- 内存：32 GiB
- Go：1.26.0 darwin/arm64
- SQLite 驱动：modernc.org/sqlite 1.53.0，`CGO_ENABLED=0` 兼容路径
- 数据库：本机临时目录、WAL、单写连接和四个只读连接

这些结果是开发机回归基线，不代表不同磁盘、虚拟化平台或容器运行时的绝对延迟。门禁使用设计文档中的固定上限，不根据机器自动放宽。

## 合成数据集

每轮性能测试在完成全部迁移的空数据库中一次性写入：

- 5 名用户，首位为管理员，其余为普通成员；密码只哈希一次。
- 10,000 部合成电影，每部具有唯一的本地 ID 和 TMDB 外部身份。
- 首位用户的 10,000 条 `completed` 状态，来源为低优先级 `external_default`。
- 50,000 条不可变观看事件和对应参与者，按一小时间隔分布并截止于 `2026-07-13T12:00:00Z`。

种子写入使用一个 SQLite 事务和预编译语句。同步测试随后创建带加密合成凭据的 Jellyfin 账户，由合成 Provider 按每页 200 条提供历史，再通过真实 Scheduler、ProviderRunner 和 `CandidateService.Ingest` 路径处理 10,000 条初次事件与 100 条增量事件。该路径包含任务领取、运行状态、凭据解密、分页、HistoryPage 校验、游标、匹配、候选持久化、观看事件、参与者、个人状态、日期投影和外部映射事务，不访问外部网络。

## 阈值与算法

| 指标 | 样本与范围 | 通过阈值 |
|---|---|---:|
| 7 月日历本地 API p95 | 初次同步期间，3 次预热后 30 个完整认证 HTTP 请求 | `< 200ms` |
| `completed` 影库筛选 p95 | 初次同步期间，3 次预热后 50 个完整认证 HTTP 请求 | `< 300ms` |
| 10,000 条初次同步 | 一次持久化 Scheduler/ProviderRunner 运行的总耗时 | `< 5min` |
| 100 条增量同步 | 延续持久化游标的下一次运行总耗时 | `< 60s` |

HTTP 测量经过真实路由、会话认证、records 服务、SQLite 查询和 JSON 编码，但使用进程内 `httptest`，不包含 TCP、反向代理或浏览器渲染。p95 使用最近邻法：将耗时升序排列，选取 `ceil(0.95 * n)` 位置。每个 HTTP 样本都验证返回条目数；同步阶段还精确验证候选、事件、参与者、映射、状态来源、run summary 和最终游标，空响应或 no-op 不能通过门禁。

| 轮次 | 日历 p95 | 影库 p95 | 初次同步 | 增量同步 |
|---:|---:|---:|---:|---:|
| 1 | 54.148416ms | 10.69275ms | 4.483806125s | 39.80025ms |
| 2 | 50.9615ms | 10.692542ms | 4.586084833s | 44.458167ms |

## 中断恢复验证

恢复脚本启动当前 Go 测试二进制的独立子进程，并在子进程明确写出 ready 信号后强制终止。覆盖以下时点：

| 中断点 | 终止时状态 | 恢复后断言 |
|---|---|---|
| 普通写入 | 新影视仍在未提交事务中 | 探针影视不存在 |
| 真实迁移事务 | `applyMigration` 已执行 DDL、尚未写入 `schema_migrations` | 探针表和版本行都不存在，正式迁移仍可重放验证 |
| 真实同步事务 | `CandidateService.Ingest` 已写多表副作用、在候选插入前阻塞 | 候选、同步事件和映射不存在；原状态、事件与参与者计数不变 |
| 备份 | SQLite Online Backup 快照完成，归档尚未提交 | 启动清理移除残留快照，没有完成或临时备份文件 |

迁移和同步分支使用仅注册到子进程新 SQLite 连接的阻塞函数，确保 SIGKILL 发生在真实生产事务的指定边界。每次终止后都会重新打开数据库、重新执行迁移校验、调用 readiness，并要求 `PRAGMA integrity_check` 返回 `ok`。原有用户、影视、状态和观看事件必须保留，未提交探针不得出现。服务启动还会清除上次强制终止留下的 `.snapshot-*`、归档 `.partial` 和恢复上传临时文件，避免反复失败耗尽数据卷。

三轮中，真实迁移中断分别耗时 0.04s、0.03s、0.02s；普通写入、真实同步和备份三个分支合计分别耗时 0.33s、0.32s、0.32s。四个中断点在每轮中均通过。
